# Step 1: Team Capacity & Sprint Setup

## MANDATORY EXECUTION RULES:

- 🛑 NEVER skip this step — capacity data is required for sprint distribution
- ✅ Ask all questions in one block
- 📋 Smart defaults based on industry standards — propose them clearly
- ✅ Speak in `{communication_language}`

## YOUR TASK:

Collect team capacity, velocity, sprint duration and start date. Check if Jira sync is configured.

---

## EXECUTION SEQUENCE:

### 1. Check Jira Mode

Read `{project-root}/_sedona/core/config.yaml` or `project-setup.md` for `pm_mode`.

- If `pm_mode: jira` → set `jira_sync: true`, load `_sedona/core/jira-config.yaml`
- Otherwise → set `jira_sync: false`

Announce:
```
**📋 Jerome:** Jira sync : {activé ✓ / désactivé}
```

### 2. Collect Capacity Info

Ask in one block:

```
**📋 Jerome:** Configurons la capacité de l'équipe pour planifier les sprints.

**[1] Durée d'un sprint**
  - A) 1 semaine
  - B) 2 semaines _(recommandé)_
  - C) 3 semaines
  - D) 4 semaines

**[2] Date de démarrage du Sprint 1**
Ex: 2026-05-05 (lundi prochain recommandé)

**[3] Taille de l'équipe**
Nombre de développeurs disponibles sur ce projet.
Ex: 1, 2, 3...

**[4] Capacité par développeur par sprint**
En story points. Si tu ne sais pas, utilise 8 SP/dev/sprint comme référence.

**[5] Vélocité connue ?**
Si tu as déjà des données de sprints passés, donne la vélocité moyenne en SP.
Sinon tape [N] — j'utiliserai la capacité théorique avec un facteur 0.8 (réaliste).

**[6] Stories points — comment tu veux les estimer ?**
  - A) Je les estime moi-même story par story
  - B) Claude les estime automatiquement (je valide après)
  - C) Toutes les stories = 3 SP (estimation flat simple)
```

**HALT — wait for user.**

### 3. Calculate Effective Velocity

Based on user input:

```
team_capacity_per_sprint = {nb_devs} × {sp_per_dev}

if velocity_known:
  effective_velocity = {user_velocity}
else:
  effective_velocity = team_capacity_per_sprint × 0.8  # realistic factor
```

Display:
```
**📋 Jerome:** Voici la capacité calculée :

- Durée sprint : {duration}
- Équipe : {nb_devs} dev(s)
- Capacité théorique : {team_capacity} SP/sprint
- Vélocité effective : {effective_velocity} SP/sprint
- Méthode d'estimation : {method}

Dates prévisionnelles :
- Sprint 1 : {start_date} → {end_date}
- Sprint 2 : {start_date+1} → {end_date+1}
- Sprint 3 : {start_date+2} → {end_date+2}
...

C'est bon ? [O]ui / [M]odifier
```

**HALT — wait for confirmation.**

### 4. Save Capacity Config

Append to `{implementation_artifacts}/sprint-config.yaml`:

```yaml
# Sprint Configuration — generated by sedona-sprint-planning
sprint_duration_weeks: {duration}
sprint_start_date: "{start_date}"
nb_developers: {nb_devs}
sp_per_dev_per_sprint: {sp_per_dev}
effective_velocity: {effective_velocity}
estimation_method: "{method}"
jira_sync: {jira_sync}
```

### 5. Continue

Load `./step-02-story-estimation.md`.

✅ Effective velocity calculated and confirmed
✅ Sprint dates projected and shown
✅ Config saved
