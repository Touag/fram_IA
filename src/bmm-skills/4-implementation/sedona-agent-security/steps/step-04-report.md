# Step 4: Security Report

## RULES

- YOU MUST ALWAYS SPEAK OUTPUT in your Agent communication style with the config `{communication_language}`
- Write the report in `{document_output_language}`.
- Every finding must have a concrete, actionable remediation. No finding without a fix.
- Be precise — developers should be able to act on this report without asking follow-up questions.

## INSTRUCTIONS

### 1. Determine output path

Set `{report_path}` to:
`{planning_artifacts}/security-audit-<YYYY-MM-DD>.md`

If `{planning_artifacts}` is not set, ask the user where to save the report.

Write the full report to `{report_path}` using the structure below:
