### 2. Gather Project Identity (Single Question Block)

Ask all of these at once — do NOT split into separate turns:

```
**🚀 Lysandre:** Bienvenue ! Répondons à 3 questions pour initialiser ton projet.

**[1] Nom du projet**
Comment s'appelle ce projet ? _(défaut : nom du dossier courant)_

**[2] Description courte**
En une phrase, c'est quoi ? _(optionnel, mais aide beaucoup les agents)_

**[3] Type de projet**
  - A) Web app / SaaS
  - B) Mobile app
  - C) API / Backend service
  - D) Design system / Component library
  - E) Data / AI product
  - F) Game
  - G) Autre

**[4] Langue de communication préférée**
  - A) Français
  - B) English
  - C) Autre (précise)
```

**HALT — wait for user responses.**

### 3. Confirm and Continue

Summarize what was gathered:

```
**🚀 Lysandre:** Parfait, voilà ce que j'ai enregistré :

- **Projet :** {project_name}
- **Description :** {project_description}
- **Type :** {project_type}
- **Langue :** {communication_language}

Ça te va ?  [O]ui / [N]on (modifier)
```

**HALT — wait for confirmation.**

If user says no, ask what to change and loop back.

### 4. Write Config (if new project)

If `_sedona/core/config.yaml` does not exist, create it:

```yaml
# Sedona Project Configuration
# Auto-generated by sedona-new on {date}

project_name: "{project_name}"
project_description: "{project_description}"
project_type: "{project_type}"
communication_language: "{communication_language}"
document_output_language: "{communication_language}"
user_skill_level: "intermediate"

```

Create directories:
```bash
mkdir -p _sedona/core
mkdir -p docs/sedona/planning-artifacts
mkdir -p docs/sedona/implementation-artifacts
```
