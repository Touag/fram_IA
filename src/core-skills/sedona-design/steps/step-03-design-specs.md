# Step 3: Design Specifications

## MANDATORY EXECUTION RULES:

- 🛑 NEVER write specs for decisions not made in step 2 — specs document decisions, they don't make them
- ✅ Specs must be implementation-ready — a developer should be able to build from this without asking questions
- 📋 Organize by token → component → layout hierarchy
- ✅ Speak in `{communication_language}`

## YOUR TASK:

Convert the design decisions from exploration into a structured, implementation-ready specification document. This is the handoff artifact.

---

```

## 4. Accessibility Checklist

- [ ] All interactive elements reachable by keyboard
- [ ] Focus indicator visible (min 3:1 contrast ratio)
- [ ] All images have alt text defined
- [ ] Color is never the sole conveyor of information
- [ ] Text contrast ≥ 4.5:1 (AA) for body text
- [ ] Text contrast ≥ 3:1 (AA) for large text (18px+ or 14px bold+)
- [ ] Touch targets ≥ 44x44px on mobile
- [ ] Motion reducible (prefers-reduced-motion support)
```


**HALT — wait for user selection.**

Handle each option:

**[R] Fill TBD values** — go through each TBD item one by one, ask for value or generate suggestion with rationale.

**[T] Generate config** — produce either:

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: { ... },
      fontFamily: { ... },
      spacing: { ... },
      borderRadius: { ... },
      boxShadow: { ... },
    }
  }
}
```

Or CSS custom properties:

```css
:root {
  --color-primary-500: ...;
  /* etc. */
}
```

**[V] Review specs** — Load `./step-04-design-review.md`.

**[N] Done** — wrap up:

