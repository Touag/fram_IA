# Step 1: Jira Configuration Setup

## MANDATORY EXECUTION RULES:

- 🛑 NEVER store the API token in plain text in any output file shown to the user — it goes only in jira-config.yaml
- ✅ Validate the connection before proceeding
- 📋 Speak in `{communication_language}`

## YOUR TASK:

Collect Jira credentials, validate the connection, write the config file.

---

## EXECUTION SEQUENCE:

### 1. Check for Existing Config

If `{jira_config_file}` exists, read it and confirm:

```
**🔗 Raphaël:** Config Jira trouvée :
- URL : {jira_url}
- Email : {jira_email}
- Projet : {jira_project_key}

On utilise ça ? [O]ui / [N]on (reconfigurer)
```

**HALT — wait for user.**

If yes → skip to step 2 (validate connection).

### 2. Collect Credentials

Ask in one block:

```
**🔗 Raphaël:** Configurons la connexion Jira. J'ai besoin de 3 infos :

**[1] URL de ton espace Jira**
Ex: https://ton-nom.atlassian.net

**[2] Email de ton compte Atlassian**
Ex: toi@email.com

**[3] API Token**
Génère-le sur : https://id.atlassian.com/manage-profile/security/api-tokens
(il ne sera visible qu'une fois — garde-le précieusement)

**[4] Clé du projet Jira**
C'est le préfixe de tes tickets. Ex: si tes tickets sont SCRUM-1, SCRUM-2 → la clé est SCRUM
```

**HALT — wait for user.**

### 3. Validate Connection

Test the connection with a GET request to the Jira API:

```
GET {jira_url}/rest/api/3/myself
Authorization: Basic base64({jira_email}:{api_token})
Content-Type: application/json
```

Use the Bash tool to run:
```bash
curl -s -o /dev/null -w "%{http_code}" \
  -u "{jira_email}:{api_token}" \
  -H "Content-Type: application/json" \
  "{jira_url}/rest/api/3/myself"
```

- If HTTP 200 → connection OK
- If HTTP 401 → wrong email or token
- If HTTP 403 → token valid but insufficient permissions
- If other → connectivity issue

Report result:
```
**🔗 Raphaël:** Connexion [✓ OK / ✗ Échec — {reason}]
```

If failed → ask user to check credentials and retry. Do NOT proceed.

### 4. Fetch Project Info

Verify the project key exists:

```bash
curl -s \
  -u "{jira_email}:{api_token}" \
  -H "Content-Type: application/json" \
  "{jira_url}/rest/api/3/project/{jira_project_key}"
```

Extract and display:
- Project name
- Project type (scrum/kanban/etc.)
- Available issue types (to confirm Epic and Story exist)

```
**🔗 Raphaël:** Projet trouvé : **{project_name}** ({project_type})
Types de tickets disponibles : {issue_types}

Epic → mapped to: {epic_issue_type}
Story → mapped to: {story_issue_type}

C'est bon ? [O]ui / [N]on
```

**HALT — wait for confirmation.**

### 5. Write Config File

Create `{jira_config_file}`:

```yaml
# Jira Configuration — generated by sedona-jira-sync
# WARNING: This file contains your API token. Add it to .gitignore !

jira_url: "{jira_url}"
jira_email: "{jira_email}"
api_token: "{api_token}"
project_key: "{jira_project_key}"
epic_issue_type: "{epic_issue_type}"
story_issue_type: "{story_issue_type}"
```

Also append `_sedona/core/jira-config.yaml` to `.gitignore` if it exists:
```bash
echo "_sedona/core/jira-config.yaml" >> .gitignore
```

```
**🔗 Raphaël:** Config sauvegardée dans `_sedona/core/jira-config.yaml`.
⚠️  Ce fichier est ajouté au .gitignore — ne le commitez jamais.
```

Load `./step-02-parse-artifacts.md`.
