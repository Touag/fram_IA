The Sedona (**B**uild **M**ore **A**rchitect **D**reams) is an AI-driven development framework module within the Sedona Ecosystem that helps you build software through the whole process from ideation and planning all the way through agentic implementation. It provides specialized AI agents, guided workflows, and intelligent planning that adapts to your project's complexity, whether you're fixing a bug or building an enterprise platform.

If you're comfortable working with AI coding assistants like Claude, Cursor, or GitHub Copilot, you're ready to get started.

## Extend and Customize

Want to expand Sedona with your own agents, workflows, or modules? The **[Sedona Builder](https://sedona-builder-docs.sedona.org/)** provides the framework and tools for creating custom extensions, whether you're adding new capabilities to Sedona or building entirely new modules from scratch.

## What You'll Need

Sedona works with any AI coding assistant that supports custom system prompts or project context. Popular options include:

- **[Claude Code](https://code.claude.com)** — Anthropic's CLI tool (recommended)
- **[Cursor](https://cursor.sh)** — AI-first code editor
- **[Codex CLI](https://github.com/openai/codex)** — OpenAI's terminal coding agent

You should be comfortable with basic software development concepts like version control, project structure, and agile workflows. No prior experience with Sedona-style agent systems is required—that's what these docs are for.
